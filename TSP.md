## Descripción

TSP (Team Software Process) es una metodología de ingeniería de software que proporciona un marco disciplinado para que los equipos produzcan software de alta calidad.

## Componentes Principales

### 1. Planificación
- Definición clara de objetivos
- Asignación de recursos
- Estimación de tiempos
- Identificación de riesgos

### 2. Diseño
- Arquitectura del sistema
- Diseño de componentes
- Interfaces y datos

### 3. Desarrollo
- Implementación del código
- Revisiones de código
- Control de calidad

### 4. Pruebas
- Pruebas unitarias
- Pruebas de integración
- Pruebas del sistema

### 5. Cierre
- Documentación final
- Lecciones aprendidas
- Entrega del proyecto
